---
title: Multi-Camera Setup
description: Use multiple Mighty cameras at the same time by assigning each camera its own USB IP subnet.
---

Mighty appears to the host computer as a USB Ethernet device. Each camera serves
the Visualizer and Mighty Protocol over a fixed USB IP address. By default, that
address is `192.168.7.1`.

If two Mighty cameras use the same USB subnet at the same time, the host cannot
route to them reliably because both devices claim the same address. Change the
USB IP range on each camera so every device has its own subnet.

## Choose one range per camera

The Visualizer can assign one of the built-in USB ranges from `7` through `14`.
The selected number becomes the third part of the address:

| Camera | USB IP range | Camera address |
| --- | --- | --- |
| Camera 1 | `7` | `http://192.168.7.1` |
| Camera 2 | `8` | `http://192.168.8.1` |
| Camera 3 | `9` | `http://192.168.9.1` |
| Camera 4 | `10` | `http://192.168.10.1` |

The camera always keeps `.1` for itself. DHCP addresses for the host start at
`.2`, so a camera on range `10` serves itself at `192.168.10.1` and gives the
host an address such as `192.168.10.2`.

## Change the range in the Visualizer

Change one camera at a time:

1. Connect only the camera you want to configure.
2. Open the Visualizer at [mightycamera.com/viz](https://mightycamera.com/viz)
   or at the camera's current USB address.
3. Wait for the connection status to show that the camera is connected.
4. Open the **Status** panel.
5. Find **USB IP Range**.
6. Pick an unused range from `7` to `14`.
7. Click **Apply**.
8. Wait for the camera to reboot.
9. Reconnect to the camera at its new address, for example
   `http://192.168.10.1`.

After every camera has a different range, connect them all to the host. Use the
connection dropdown in the Visualizer status badge to choose which camera
endpoint to view or control.

## Example

For two cameras, leave the first camera on the default range and move the second
camera to range `8`:

| Device | Address after setup |
| --- | --- |
| First camera | `http://192.168.7.1` |
| Second camera | `http://192.168.8.1` |

For three cameras, use `7`, `8`, and `9`. Continue assigning one unique range
per camera.

## Synchronizing camera shutter

For stereo or any multi-camera capture where frames must represent the same
instant, synchronize the shutter trigger.

For a two-camera synced setup:

1. Keep one camera on the default USB IP range, `192.168.7.1`.
2. Change the other camera to USB IP range `8`, so it is reachable at
   `192.168.8.1`. You can do this from the Visualizer at
   [mightycamera.com/viz](https://mightycamera.com/viz).
3. Wire the FSYNC pins of both cameras together.
4. Drive that shared FSYNC line from the PWM pin of one camera. For example,
   connect both cameras' FSYNC pins to camera 1's PWM pin.
5. Enable SYNC mode on both cameras from the Visualizer.

After this, both cameras should expose separate USB IP addresses while sharing
the same shutter trigger. The synced stereo example aligns the device timestamp
bases in software and then renders matched image pairs from the shared-shutter
stream.

## Stereo calibration

Stereo depth needs a calibrated camera pair mounted rigidly with a fixed
baseline. The calibration estimates each camera's intrinsics and the rigid
transform between the two cameras. If you also record IMU topics, the
calibration report can include the camera-to-IMU chain. For testing, you can use
the example mount at
`mighty-web/mighty-protocol/examples/python/synced_stereo/stereo-mount.stl`.

To create a calibration bag:

1. Finish the hardware setup above: unique IP addresses, shared FSYNC/PWM
   wiring, and SYNC mode enabled on both cameras.
2. Mount the cameras rigidly with a fixed baseline.
3. Run the synced stereo Python example without depth enabled:

   ```bash
   python3 mighty-web/mighty-protocol/examples/python/synced_stereo/main.py \
     --left-host http://192.168.7.1 \
     --right-host http://192.168.8.1
   ```

4. Open the calibration grid at
   [mightycamera.com/calib](https://mightycamera.com/calib) and put it in full
   screen.
5. Point both cameras at the AprilGrid.
6. Click **Start Rec** in the synced stereo app.
7. Move the rigid stereo rig through the calibration motions while keeping the
   grid visible to both cameras.
8. Click **Stop Rec** when done.
9. On the `/calib` page, measure and enter the correct AprilGrid width first.
10. Open **Advanced** and set **Camera topics** to
    `/cam0/image_raw/compressed /cam1/image_raw/compressed`.
11. Manually upload the recorded bag from the synced stereo app as a custom ROS
   bag.
12. Start calibration from that custom bag. It should take about 15 minutes.
13. Download the report and find the generated `-camchain-imucam.yaml` file.

Use that `-camchain-imucam.yaml` file as the calibration input for stereo depth.

## Stereo depth example

The synced stereo Python example can display both camera streams, IMU traces,
matched timestamps, and rectified SGBM metric depth in one OpenCV window.

After calibration, run the example with the downloaded
`-camchain-imucam.yaml`:

```bash
python3 mighty-web/mighty-protocol/examples/python/synced_stereo/main.py \
  --left-host http://192.168.7.1 \
  --right-host http://192.168.8.1 \
  --depth /path/to/your-camchain-imucam.yaml \
  --depth-scale 0.5 \
  --depth-min-m 0.2 \
  --depth-max-m 5.0
```

You can also replay a previously recorded stereo bag through the same viewer and
depth pipeline:

```bash
python3 mighty-web/mighty-protocol/examples/python/synced_stereo/main.py \
  --bag /path/to/synced_stereo.bag \
  --depth /path/to/your-camchain-imucam.yaml \
  --depth-scale 0.5 \
  --depth-min-m 0.2 \
  --depth-max-m 5.0
```

Bag playback expects the image topics
`/cam0/image_raw/compressed /cam1/image_raw/compressed` and IMU topics
`/imu0 /imu1`, matching bags recorded by the synced stereo example.

The depth view uses the same rendered synced stereo pair as the raw preview. It
rectifies the double-sphere cameras from the Kalibr YAML, computes SGBM
disparity, and converts that disparity to metric depth using:

```text
Z = focal_px * baseline_m / disparity_px
```

Depth colors are fixed-range metric colors, not per-frame relative colors.
`--depth-min-m` maps to the nearest/hottest color and `--depth-max-m` maps to
the farthest/coolest color. Invalid depth is shown dark.

## Reset to the default range

If you lose track of a camera's range, hold the camera button for at least 10
seconds, then release it. This resets the USB IP range to the default
`192.168.7.x` range and reboots the camera.

After a range change or reset, the host operating system may need a few seconds
to refresh its USB Ethernet lease. If the new address does not respond
immediately, unplug and reconnect the camera.
